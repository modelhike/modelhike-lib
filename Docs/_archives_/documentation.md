## 🔍 How ModelHike Differs from Traditional Model-Based Software Development

| Aspect                   | Model-Based Software Dev (MBSD)    | **ModelHike**                                         |
| ------------------------ | ---------------------------------- | ----------------------------------------------------- |
| **Philosophy**           | Top-down formal modeling           | Developer-first, iterative modeling                   |
| **Primary Audience**     | Enterprise architects, UML users   | Product engineers, full-stack devs                    |
| **Modeling Language**    | Often graphical (UML, SysML)       | **Markdown-inspired DSL**                             |
| **Developer Experience** | Tool-heavy, abstracted from code   | Git-native, code-near, joy-focused                    |
| **Tooling**              | Often proprietary and heavyweight  | Open, minimal, CLI + editor friendly                  |
| **Execution Flow**       | Model → Code generation → Validate | Prompt → DSL → GenAI/Template output → Iterate        |
| **Flexibility**          | Rigid modeling standards           | Freeform, composable modeling                         |
| **Business Logic**       | Often external to models           | Embedded as sequence/data flow in **ModelHike DSL**   |
| **GenAI Support**        | Rare to none                       | **Deeply integrated** with GenAI in loop              |
| **Goal**                 | Formal system specification        | **Flow, clarity, context** in real-world app building |

## 7. 🤔 Campfire FAQs

_Your “aha” corner_ (Your Questions Answered)

- **How do I update a frozen template?**
  Edit the model or un-accept the file; regeneration goes back to AI if needed, or tweak the template directly.
- **Can I target any language or framework?**
  Yes—ModelHike is stack-agnostic. Provide a template engine for your chosen tech, and you’re set.
- **What if AI output drifts over time?**
  Re-prompt, refine your model, or re-freeze the desired version. You retain full control.
